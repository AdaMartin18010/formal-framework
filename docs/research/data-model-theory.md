# 数据模型理论与实践

## 理论基础

- 形式化描述实体、关系、约束、索引等数据库结构
- 支持多种数据库（PostgreSQL、MySQL、SQLite等）
- 强调数据一致性、完整性、可迁移性

## 关键概念

- **实体关系建模**：ER模型、表结构、主外键
- **约束与校验**：唯一性、非空、正则、外键
- **迁移与版本管理**：数据库演进、自动迁移脚本
- **索引与性能**：索引策略、查询优化

## 实践方法

- 统一DSL描述数据结构
- 自动生成DDL、迁移脚本
- 支持多数据库适配与切换
- 集成数据校验与测试

## 示例

```yaml
entities:
  - name: "User"
    fields:
      - name: "id"
        type: "uuid"
        primary_key: true
```
