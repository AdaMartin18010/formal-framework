# 专家评审体系 (Expert Review System)

> 本文档定义形式化框架文档与模型产物的专家评审流程、角色与记录要求。

## 1. 目的

- 确保理论、行业案例、L2/L3/L4 文档在**学术严谨性**与**工程可用性**上达到可发布水平。
- 为完成度等级 **RC** 与 **FINAL** 提供可追溯的评审依据。

## 2. 评审类型

| 类型 | 适用对象 | 要求 |
|------|----------|------|
| **技术评审** | theory.md、dsl-draft.md、行业案例、L3 标准模型 | 至少 1 名具备形式化方法或对应行业经验的评审人 |
| **标准对齐评审** | 与 ISO/IEC、IEEE、CNCF 等对标的文档 | 评审人熟悉相关标准或官方文档 |
| **学术评审** | 含形式化定义、公理、可证明性论述的文档 | 适用于目标为 FINAL 的文档；可与技术评审合并为「双评审」 |

## 3. 评审检查项

- **引用合规**：关键结论是否遵循 [CITATION_STANDARDS.md](CITATION_STANDARDS.md)，具备高可信引用。
- **结构完整**：是否包含术语/映射/方法/DSL/验证/案例/引用等适用维度（见 [QUALITY_STANDARDS.md](QUALITY_STANDARDS.md)）。
- **术语一致**：是否与全局术语表、概念索引一致。
- **链接有效**：文内与交叉引用、证据条目链接可访问。
- **图表规范**：Mermaid/表格符合项目规范。
- **证据条目**：新增标准/课程/应用映射是否已创建或更新 evidence 条目。
- **完成度等级**：DRAFT/RC/FINAL 标识是否与内容及门禁项一致。

## 4. 评审记录

- 每次评审须留下**可追溯记录**：评审人（或角色）、日期、结论（通过/驳回/修订）、主要意见或修改建议。
- 记录可存放在：证据条目的「评审」小节、文档末尾的「评审历史」、或集中评审日志（如 `docs/review-log/`）。
- **RC**：至少一次专家评审通过。**FINAL**：建议学术+工程双评审通过。

## 5. 与质量门禁的衔接

- 未通过评审的文档不应标为 RC 或 FINAL；可保留 DRAFT 并标注「待评审」或「修订中」。
- 与 [QUALITY_STANDARDS.md](QUALITY_STANDARDS.md)、[CITATION_STANDARDS.md](CITATION_STANDARDS.md)、[community-framework.md](community-framework.md) 第 3.0 节共同构成 PR 与发布门禁。

## 6. 实施建议

- 在 PR 模板中增加「专家评审」选项或清单，便于维护者勾选与记录。
- 可设立评审轮值或领域负责人（如形式化方法、云原生、金融等），确保覆盖与可持续。
