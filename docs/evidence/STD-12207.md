## 1. 基本信息

- **标准编号**：ISO/IEC/IEEE 12207
- **名称**：Systems and software engineering — Software life cycle processes
- **版本/日期**：ISO/IEC/IEEE 12207:2017 为当前发布版；FDIS 新版（Edition 2）处于批准阶段，将替代 2017 版。
- **范围**：定义软件生命周期过程，包括获取、供应、开发、运行、维护等，适用于软件及含软件的系统。
- **与本框架映射**：L3_D09 CI/CD、practice-guides、L2/L3 流程与生命周期阶段。

## 2. 摘要

- 12207 规定软件生命周期各阶段的过程与活动，与本框架 L3_D09（CI/CD 标准模型）中的流水线、阶段、触发、质量门禁等对应；practice-guides 中的方法论与生命周期规范可与之对齐。
- 关键对齐点：开发过程 ↔ L3 各标准模型（需求→设计→实现→测试→部署）；维护与运行 ↔ L3_D05 部署、L3_D06 监控。

## 3. 对齐维度

- **术语对齐**：软件生命周期过程 (software life cycle processes) ↔ 本框架「生命周期」「流程」「阶段」；开发过程 (development process) ↔ L3 开发相关标准模型。
- **结构/接口**：过程类别（主要过程、支持过程、组织过程）可与本框架 L2 元模型、L3 标准模型按阶段映射。
- **约束/不变式**：各过程输入/输出与责任主体在标准中定义，本框架 QUALITY_STANDARDS、完成度等级可与之对照。
- **度量/指标**：标准涉及过程能力与绩效；本框架 L3_D06 监控、L3_D08 测试提供可度量维度。

## 4. 映射

- **L2**：L2 各元模型对应生命周期中的「需求/设计/实现/验证」抽象；无单文件锚点，见 [docs/README](../README.md) 模型体系。
- **L3**：[L3_D09 CI/CD 标准模型](../L3_D09_CICD标准模型.md)、[L3_D08 测试标准模型](../L3_D08_测试标准模型.md)、[L3_D05 部署标准模型](../L3_D05_部署标准模型.md)。
- **L4**：各行业 L4 索引中的流程与合规要求可追溯至 12207 过程。
