## 1. 基本信息

- **项目**：Kubernetes (K8s)
- **定位**：容器编排平台；CNCF 毕业项目；L4_D90 云原生行业索引核心对标对象。
- **与本框架映射**：L3_D04 运行时标准模型（容器、Pod、Service、Ingress、Controller、调度）；L3_D05 部署（配置、声明式）；L3_D07 控制调度（kube-scheduler）；L4_D90 云原生行业索引。详细案例与组件映射见 [CN-K8S-BASE](CN-K8S-BASE.md) 及 [L4_D90_CN 行业索引与对标](../L4_D90_CN_行业索引与对标.md) §4、§6.1。

## 2. 摘要

- Kubernetes 提供 Pod/Service/Ingress/Controller/CRD 等抽象，与本框架 L3_D04 运行时标准模型中的 Workload、Service、Network、Orchestration 对象对应；kube-scheduler 与 L3_D07 控制调度标准模型对应。证据条目 CN-K8S-BASE 侧重基础编排案例；本条目 (K8S-001) 为 L4 映射矩阵与行业 README 中 evidence:K8S-001 的统一点，便于引用与审稿。

## 3. 对齐维度

- **术语对齐**：Pod ↔ L3_D04 Workload/Pod；Service ↔ L3_D01 交互 + L3_D04 服务发现；Ingress ↔ L3_D01 入口；Controller ↔ L3_D04 控制器；CRD ↔ L2 元模型扩展。
- **结构/接口**：Kubernetes API 与 L3_D01 交互标准模型（API 契约）、L3_D04 编排对象对应。
- **约束/不变式**：ReplicaBounds、EndpointHealth、ResourceLimitsSLA 等见 [alignment-L2-L3-matrix](../formal-model/alignment-L2-L3-matrix.md) §2.3。
