## 1. 基本信息

- **标准编号**：ISO/IEC/IEEE 42010（原 IEEE 1471 演进）。
- **名称**：Software, systems and enterprise — Architecture description
- **版本/日期**：**42010:2022**（Edition 2，2022 年 11 月发布，现行）；旧版 42010:2011 已取代。
- **范围**：规定软件、系统、企业及系统之系统的架构描述要求，包括架构描述框架 (ADF)、架构描述语言 (ADL)、视角 (viewpoint)、视图 (view)、模型种类 (model kind) 等。
- **与本框架映射**：L3 交互/运行时/部署等标准模型中的架构视图与视角；L2 元模型作为架构描述的抽象层次。

## 2. 摘要

- **架构 vs 架构描述**：42010 明确区分「架构」（系统本身的关键决策与结构）与「架构描述」（表达该架构的制品，如文档、视图、模型）；本框架 L2/L3 文档与模型即架构描述，不规定具体实现。
- 42010 的「架构描述」「视图」「视角」与本框架 L3 标准模型（交互、数据、功能、运行时、部署、监控、测试、CI/CD、分布式模式）一一对应为多视图架构；L2 元模型对应架构描述中的抽象概念层。
- 关键对齐点：视角 (viewpoint) ↔ 本框架按模型类型（数据/功能/交互/运行/部署/监控等）划分的视角；视图 (view) ↔ 各 L3 文档与 industry-model 中的具体架构描述。

## 3. 对齐维度

- **术语对齐**：架构描述 (architecture description) ↔ 本框架 L2/L3/L4 文档体系；视图/视角 ↔ L3 各标准模型及行业 README 中的架构图与表格。
- **结构/接口**：利益相关者—关注点—视角—视图 链与本框架「目标受众—学习路径—模型层次」可对照。
- **约束/不变式**：架构描述应满足的完整性、一致性要求与本框架 QUALITY_STANDARDS、交叉引用一致。
- **度量/指标**：架构决策与 Rationale 可对应本框架 theory.md、dsl-draft.md 中的设计理由与验证。

## 4. 映射

- **L2**：L2 八个元模型（交互、数据、功能、运行时、部署、监控、控制调度、测试）对应架构描述中的概念/元模型层。
- **L3**：L3 十个标准模型为架构视图的具体化，见 [L3_D01 交互标准模型](../L3_D01_交互标准模型.md)～[L3_D10 分布式模式标准模型](../L3_D10_分布式模式标准模型.md)。
- **L4**：各 [industry-model](../industry-model/) README 与 theory 为行业视角下的架构描述。
