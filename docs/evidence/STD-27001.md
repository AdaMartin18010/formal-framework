## 1. 基本信息

- **标准编号**：ISO/IEC 27001
- **名称**：Information security management systems — Requirements
- **版本/日期**：ISO/IEC 27001:2022（当前版）。
- **范围**：规定建立、实施、保持并持续改进信息安全管理体系（ISMS）的要求，包括风险评估与处置、安全控制等。
- **与本框架映射**：L2_D05 部署（配置与基础设施安全）、L2_D06 监控（审计与安全监控）；行业模型中的合规与安全要求。

## 2. 摘要

- 27001 的 ISMS 要求与本框架「部署模型」中的配置管理、访问控制、安全策略对应；「监控模型」中的审计日志、安全事件与 27001 的监控与测量、审计要求对应。
- 关键对齐点：安全控制（如 A.5～A.8）↔ 部署与运行中的访问控制、加密、备份；审计 ↔ L3_D06 监控与审计能力。

## 3. 对齐维度

- **术语对齐**：信息安全管理体系 (ISMS) ↔ 本框架部署与监控中的安全策略与审计；风险评估 ↔ 质量与合规度量。
- **结构/接口**：27001 附录 A 控制项可与 L3_D05 部署配置、L3_D06 监控告警与日志结构对照。
- **约束/不变式**：合规要求与本框架 QUALITY_STANDARDS、行业 L4 合规（如金融 PCI-DSS）可联合使用。
- **度量/指标**：安全绩效指标与 L3_D06 监控指标、证据条目中的合规验证一致。

## 4. 映射

- **L2**：[L2_D05 部署元模型](../L2_D05_部署元模型.md)（配置、基础设施、安全相关）；[L2_D06 监控元模型](../L2_D06_监控元模型.md)（审计、日志、告警）。
- **L3**：[L3_D05 部署标准模型](../L3_D05_部署标准模型.md)、[L3_D06 监控标准模型](../L3_D06_监控标准模型.md)。
- **L4**：金融 L4_D91（PCI-DSS 等）、各行业安全与合规证据条目。
